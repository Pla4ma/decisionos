## Implementation Guidelines

### Parking Rules

1. **No Early Implementation** - Features remain parked until designated stage
2. **MVP Focus** - Current development stays on core MVP functionality
3. **Regular Review** - Parking lot reviewed each planning cycle
4. **User Feedback Priority** - User requests may reprioritize parked features
5. **Technical Debt Awareness** - Consider technical implications when unparking

### Unparking Criteria

Features may be unparked when:
- Designated stage is reached
- Critical user need emerges
- Technical dependencies are satisfied
- Resource capacity allows
- MVP core is stable and complete
